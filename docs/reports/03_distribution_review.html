Probability models and diagnostics for the World Cup modeling workflow

These original project notes are influenced by public NC State statistics teaching resources: short conceptual framing, visible math, reproducible R/Quarto code, and simulation/diagnostic thinking.

1 Purpose

Each variable has an explicit statistical shape. Soccer data mixes low-count outcomes, ordered categorical results, probabilities, text/news counts, player availability flags, and continuous context variables.

The workflow lists plausible distributions, builds diagnostics, and filters models that do not fit the data or the betting question.

7 Modeling Implications

The current diagnostics point to these practical choices:

  • Use Poisson and negative binomial families for goals, cards, injuries, and news-count features because these targets are event counts.
  • Keep Bernoulli, binomial, and beta thinking around win flags, scoring flags, form-window summaries, and probability calibration.
  • Use gamma, exponential, and geometric ideas for rest, event timing, and scoring drought features, but only after they improve validation metrics.
  • Use normal, t, chi-squared, and F distributions mainly for diagnostics: residual shape, autocorrelation, heteroskedasticity, goodness of fit, and variance shifts.
  • Use uniform draws for Monte Carlo simulation and randomized validation, not as a claim about match outcomes.

8 Review Notes Captured

The public materials reviewed emphasize a few habits that fit this project well:

  • Use computing to build intuition, not just final answers.
  • Put distribution assumptions beside visuals so model choices are auditable.
  • Compare models on held-out or cross-validated performance, not training fit alone.
  • Teach the reader the data grain before showing the model.
  • Keep source code reproducible while making the public page readable.

Sources reviewed include Professor Justin Post’s personal/course site, his ST 558 course repository, his Shiny teaching app page, and the Distributome project linked from those teaching resources.

9 Model Quality Checklist

  1. Goals distribution diagnostics by tournament era and confederation.
  2. Mean-versus-variance checks for count features.
  3. Calibration plots for ordinal result probabilities.
  4. Cross-validation by time split.
  5. Predicted versus observed W/D/L rates by probability bucket.
  6. Residual autocorrelation checks after every major model refresh.
  7. Distribution-specific tests before promoting a challenger model.

  and scoring drought features, but only after they improve validation metrics.
- Use **normal**, **t**, **chi-squared**, and **F** distributions mainly for
  diagnostics: residual shape, autocorrelation, heteroskedasticity, goodness of
  fit, and variance shifts.
- Use **uniform** draws for Monte Carlo simulation and randomized validation,
  not as a claim about match outcomes.

## Review Notes Captured

The public materials reviewed emphasize a few habits that fit this project well:

- Use computing to build intuition, not just final answers.
- Put distribution assumptions beside visuals so model choices are auditable.
- Compare models on held-out or cross-validated performance, not training fit alone.
- Teach the reader the data grain before showing the model.
- Keep source code reproducible while making the public page readable.

Sources reviewed include Professor Justin Post's personal/course site, his ST 558 course repository, his Shiny teaching app page, and the Distributome project linked from those teaching resources.
