Expanded historical model comparison for goals and match results.

Challenger model comparison for the World Cup forecasting system, including count models, smooth models, ordinal and multinomial classifiers, stepwise regression, tree models, and feature importance.

1 Purpose

The production forecast should not change just because a model sounds more advanced. This page compares challenger models against the current regression stack before any promotion into the public prediction cards.

Current challengers:

  • Stepwise OLS: an interpretable regression that selects terms by AIC.
  • Count models: Poisson, quasi-Poisson, negative binomial, and two-stage zero-aware goal models.
  • Smooth models: a GAM Poisson model that can learn curved relationships.
  • Result classifiers: ordinal logistic, multinomial logistic, regularized multinomial, classification tree, and random forest result models.
  • Expanded population sweep: rolling form, opponent form, head-to-head history, rest, experience, tournament context, GBM, random forest, SVM, and shrinkage baselines.
  • XGBoost goals: a gradient-boosted count model when the local xgboost package is installed.
  • Random forest goals: a free local tree fallback while XGBoost is not installed.

How to read the tables:

  • RMSE is the average size of the goal prediction error. Lower is better.
  • MAE is the average absolute goal error. Lower is better.
  • Brier score measures probability quality for win, draw, and loss forecasts. Lower is better.
  • Log loss heavily penalizes confident wrong probabilities. Lower is better.
  • Accuracy is the share of matches where the top predicted result matched the actual result. Higher is better.
  • Time split is the stricter validation check because newer matches are held out from older training data.

10 Interpretation

The challenger models are useful for two separate reasons:

  • Stepwise regression helps simplify the equation while staying interpretable.
  • Tree models can capture nonlinear interactions without hand-writing every term.
  • Count models better match the fact that goals are non-negative integers.
  • Smooth models can improve accuracy when the relationship between team strength and goals is curved rather than straight.
  • Multinomial and ordinal models give direct win, draw, and loss probabilities.

The tree results should be treated as a candidate signal, not an automatic upgrade. Before promotion, the next pass should compare calibration, tournament backtesting, knockout advancement behavior, and stability across multiple time splits.

## Purpose

The production forecast should not change just because a model sounds more advanced.
This page compares challenger models against the current regression stack before any
promotion into the public prediction cards.

Current challengers:

- **Stepwise OLS:** an interpretable regression that selects terms by AIC.
- **Count models:** Poisson, quasi-Poisson, negative binomial, and two-stage
  zero-aware goal models.
- **Smooth models:** a GAM Poisson model that can learn curved relationships.
- **Result classifiers:** ordinal logistic, multinomial logistic, regularized
  multinomial, classification tree, and random forest result models.
- **Expanded population sweep:** rolling form, opponent form, head-to-head
  history, rest, experience, tournament context, GBM, random forest, SVM, and
  shrinkage baselines.
- **XGBoost goals:** a gradient-boosted count model when the local `xgboost`
  package is installed.
- **Random forest goals:** a free local tree fallback while XGBoost is not
  installed.

How to read the tables:

- **RMSE** is the average size of the goal prediction error. Lower is better.
- **MAE** is the average absolute goal error. Lower is better.
- **Brier score** measures probability quality for win, draw, and loss forecasts.
  Lower is better.
- **Log loss** heavily penalizes confident wrong probabilities. Lower is better.
- **Accuracy** is the share of matches where the top predicted result matched
  the actual result. Higher is better.
- **Time split** is the stricter validation check because newer matches are held
  out from older training data.
