2 Model Iterations

The model section is organized around interpretable iterations. Each iteration names the prediction target, shows the equation, explains the coefficients in plain English, and lists the checks that determine whether the model is trustworthy.

2.1 1. Simple OLS Goals Model

Question: how many goals should a team score in one match?

The simple goals model starts with team strength difference:

\[ \text{Goals} = \beta_0 + \beta_1\text{Strength Difference} + \epsilon \]

In words, beta_1 is the average change in expected goals for a one-unit increase in strength difference.

Why start here: OLS is easy to explain, fast to diagnose, and gives a clear benchmark before count models.

Main issues to check:

  • Goals are counts, so OLS can predict impossible values below 0.
  • Soccer scores are low and skewed, so residuals may not look normal.
  • Error variance can change by team strength or opponent strength.
  • Team-match rows are not fully independent because each match creates two rows.
  • Missing roster, injury, rest, venue, and market data can bias coefficients.

2.2 2. Three-Outcome Result Model

Question: what is the probability of win, draw, or loss?

The first three-outcome model is an ordinal logistic regression because the team result has a natural order:

\[ \text{loss} < \text{draw} < \text{win} \]

The model uses cumulative logits:

\[ \text{logit}\{P(\text{Result} \leq k)\} = \theta_k - \eta \]

with:

\[ \eta = \beta_1\text{Strength Difference} + \beta_2\text{Roster Strength} + \beta_3\text{Rest} + \beta_4\text{Venue} \]

The output is three probabilities:

\[ P(\text{loss}), \quad P(\text{draw}), \quad P(\text{win}) \]

Main issues to check:

  • The proportional-odds assumption may be too strong.
  • Draws are not simply halfway between loss and win.
  • A multinomial logit may fit better if win, draw, and loss behave as separate categories.
  • Calibration matters more than raw accuracy for betting comparison.
  • Market odds should be the benchmark once API odds are available.

3 Dictionary

Use this as the quick reference for acronyms and modeling terms that appear across the site.

Term Plain-English Meaning Why It Matters Here
Y or target The thing the model is trying to predict. For the goals model, Y is goals scored. For the result model, Y is win, draw, or loss.
X or predictor An input used to make the prediction. Team strength, opponent strength, rest, venue, injuries, and odds can all be predictors.
Regression A model that estimates how a target changes with predictors. It gives a structured way to turn soccer information into expected goals or probabilities.
OLS Ordinary Least Squares. A line-fitting method that chooses coefficients by minimizing squared prediction errors. It is the first goals benchmark because it is easy to explain and diagnose.
RMSE Root Mean Squared Error. A typical prediction miss, measured in the same units as the target. If goals RMSE is 1.46, the model is often off by roughly 1 to 2 goals, with big misses punished more. Lower is better.
Coefficient The estimated effect of one predictor, holding the other predictors fixed. A positive strength coefficient means stronger teams are expected to score more, all else equal.
Residual Observed value minus predicted value. Residual plots show whether the model is missing patterns.
Logit A transformation that converts a probability into log odds. Logistic models use logits so probabilities stay between 0 and 1.
Odds Probability of an event divided by probability of it not happening. Model probabilities can be compared with betting-market odds after converting both to the same scale.
Ordinal logistic regression A logistic model for ordered outcomes. It fits loss < draw < win while producing probabilities for all three outcomes.
Multinomial logistic regression A logistic model for three or more categories without assuming an order. It is the backup if draws behave like their own special category rather than the middle of loss and win.
Accuracy Share of predictions where the top predicted class matches the actual result. Useful for a quick check, but not enough for betting because probability quality matters too.
Calibration Whether predicted probabilities match real frequencies. If the model says 60% win chance, well-calibrated teams should win about 60% of those cases.
Log loss A probability score that penalizes confident wrong predictions. Better than accuracy when the model outputs win/draw/loss probabilities. Lower is better.
Brier score Average squared error between predicted probabilities and actual outcomes. Another way to judge probability quality. Lower is better.
Test set Data held out from fitting and used only to evaluate performance. It helps estimate how the model performs on games it has not already seen.
Overfitting When a model learns noise in past data instead of repeatable signal. A model can look good historically and still fail on new World Cup matches.
Data grain What one row means in a dataset. The main modeling grain is one team in one match. One match creates two team-match rows.
API Application Programming Interface. A structured way to pull data from a provider. API-Football is the primary live football data source.

This project builds a reproducible data and modeling workflow for the 2026 FIFA World Cup.

The workflow has four layers:

  • Data collection: match history, Elo ratings, 2026 venues, weather, player metadata, odds, and news signals.
  • Data storage: local files and DuckDB tables for modeling.
  • Modeling: interpretable models in R, with Python handling source refreshes and heavier data jobs.
  • Publishing: Quarto renders a shareable GitHub Pages site from the local analysis.
