How it works

Static Pipeline

The public site is cheap and stable because all expensive work happens before the page is published.

1. Pull data offlineAPIs and public sources are refreshed by scripts, not by the browser.
2. Build model filesPython, R, SQL, and DuckDB create summarized forecast outputs.
3. Render static pagesQuarto writes public HTML into docs/ for GitHub Pages.
4. Keep raw data privateAPI keys, credentials, raw feeds, and private files are not published.
Scope: Keep this as a static report for now. RMarkdown/Quarto is enough for the public page. R Shiny can be added later for private model exploration, but it should not become the public default until the static version stops being enough.

Forecasts are probabilistic and should be used for research, discussion, and model review. They are not financial advice or guaranteed betting edges.
