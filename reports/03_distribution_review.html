Probability models and diagnostics for the World Cup modeling workflow

These original project notes are influenced by public NC State statistics teaching resources: short conceptual framing, visible math, reproducible R/Quarto code, and simulation/diagnostic thinking.

1 Purpose

Each variable has an explicit statistical shape. Soccer data mixes low-count outcomes, ordered categorical results, probabilities, text/news counts, player availability flags, and continuous context variables.

The workflow lists plausible distributions, builds diagnostics, and filters models that do not fit the data or the betting question.

if (nrow(goals_frame) > 0 && "y_goals_for" %in% names(goals_frame) && requireNamespace("ggplot2", quietly = TRUE)) {
  lambda_hat <- mean(goals_frame$y_goals_for, na.rm = TRUE)
  observed <- goals_frame |>
    dplyr::count(y_goals_for, name = "observed_rows") |>
    dplyr::mutate(observed_share = observed_rows / sum(observed_rows))
  poisson_ref <- data.frame(
    y_goals_for = seq(0, max(goals_frame$y_goals_for, na.rm = TRUE)),
    poisson_share = stats::dpois(seq(0, max(goals_frame$y_goals_for, na.rm = TRUE)), lambda = lambda_hat)
  )

  ggplot2::ggplot(observed, ggplot2::aes(x = y_goals_for, y = observed_share)) +
    ggplot2::geom_col(fill = "#157878", alpha = 0.82, width = 0.75) +
    ggplot2::geom_line(data = poisson_ref, ggplot2::aes(y = poisson_share), color = "#f2a65a", linewidth = 1.1) +
    ggplot2::geom_point(data = poisson_ref, ggplot2::aes(y = poisson_share), color = "#f2a65a", size = 2) +
    ggplot2::scale_y_continuous(labels = scales::percent_format(accuracy = 1)) +
    ggplot2::labs(
      title = "Observed goals versus simple Poisson reference",
      x = "Goals scored by team",
      y = "Share of team-match rows"
    ) +
    ggplot2::theme_minimal(base_size = 12)
} else {
  plot.new()
  text(0.5, 0.5, "Goal distribution plot is generated locally.")
}

Match resultOrdinal logistic, multinomial logistic, adjacent-category models, paired-score simulation.Useful for win/draw/loss odds comparison.
Player and news countsPoisson or negative binomial with exposure offsets where needed.Useful for goals, mentions, injuries, cards, and appearances.
ProbabilitiesBeta, Dirichlet, calibration curves, Brier/log-loss summaries.Useful once predictions are compared with market-implied probabilities.
Continuous contextNormal, t, lognormal, gamma, empirical splines, or transformed variables.Useful for weather, travel, rest, Elo differences, and player minutes.
Latent team strengthGaussian random effects, dynamic state-space ratings, hierarchical priors.Useful when teams have uneven schedule strength and sparse recent games.

4 Review Notes Captured

The public materials reviewed emphasize a few habits that fit this project well:

  • Use computing to build intuition, not just final answers.
  • Put distribution assumptions beside visuals so model choices are auditable.
  • Compare models on held-out or cross-validated performance, not training fit alone.
  • Teach the reader the data grain before showing the model.
  • Keep source code reproducible while making the public page readable.

Sources reviewed include Professor Justin Post’s personal/course site, his ST 558 course repository, his Shiny teaching app page, and the Distributome project linked from those teaching resources.

if (nrow(goals_frame) > 0 && "y_goals_for" %in% names(goals_frame)) {
  goal_counts <- table(goals_frame$y_goals_for)
  goal_mean <- mean(goals_frame$y_goals_for, na.rm = TRUE)
  goal_var <- stats::var(goals_frame$y_goals_for, na.rm = TRUE)
  zero_share <- mean(goals_frame$y_goals_for == 0, na.rm = TRUE)

  cat(metric_cards(data.frame(
    label = c("Mean Goals", "Goal Variance", "Zero-Goal Share", "Max Goals"),
    value = c(
      fmt_number(goal_mean, 3),
      fmt_number(goal_var, 3),
      fmt_percent(zero_share, 1),
      fmt_integer(max(goals_frame$y_goals_for, na.rm = TRUE))
    ),
    note = c("team-match rows", "compare with mean", "possible count-model clue", "tail check")
  )))
}
