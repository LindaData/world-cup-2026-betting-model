2 Model Iterations

The model section is organized around interpretable iterations. Each model card names the prediction target, explains the method, and lists the checks that determine whether the model is trustworthy.

2.1 1. Simple OLS Goals Model

Question: how many goals should a team score in one match?

The simple goals model starts with team strength difference:

\[ \text{Goals} = \beta_0 + \beta_1\text{Strength Difference} + \epsilon \]

In words, beta_1 is the average change in expected goals for a one-unit increase in strength difference.

Why start here: OLS is easy to explain, fast to diagnose, and gives a clear benchmark before count models.

Main issues to check:

  • Goals are counts, so OLS can predict impossible values below 0.
  • Soccer scores are low and skewed, so residuals may not look normal.
  • Error variance can change by team strength or opponent strength.
  • Team-match rows are not fully independent because each match creates two rows.
  • Missing roster, injury, rest, venue, and market data can bias coefficients.

2.2 2. Poisson Goals Model

Question: how many goals should a team score when the target is a count?

The Poisson model predicts a positive expected-goals number:

\[ \log(\lambda) = \beta_0 + \beta_1\text{Team Elo} + \beta_2\text{Opponent Elo} + \beta_3\text{Expected Result} + \beta_4\text{Match Context} \]

In words, lambda is expected goals. This is a better shape for soccer scoring than OLS because goals cannot be negative.

Main issues to check:

  • The basic Poisson assumes the mean and variance are tied together.
  • Soccer can be overdispersed, meaning the variance may be larger than the mean.
  • A negative binomial model may be needed if Poisson is too tight.
  • Forward-time validation matters more than random validation for future matches.

2.3 3. Three-Outcome Result Model

Question: what is the probability of win, draw, or loss?

The first three-outcome model is an ordinal logistic regression because the team result has a natural order:

\[ \text{loss} < \text{draw} < \text{win} \]

The model uses cumulative logits:

\[ \text{logit}\{P(\text{Result} \leq k)\} = \theta_k - \eta \]

with:

\[ \eta = \beta_1\text{Strength Difference} + \beta_2\text{Roster Strength} + \beta_3\text{Rest} + \beta_4\text{Venue} \]

The output is three probabilities:

\[ P(\text{loss}), \quad P(\text{draw}), \quad P(\text{win}) \]

Main issues to check:

  • The proportional-odds assumption may be too strong.
  • Draws are not simply halfway between loss and win.
  • A multinomial logit may fit better if win, draw, and loss behave as separate categories.
  • Calibration matters more than raw accuracy for betting comparison.
  • Market odds should be the benchmark once API odds are available.

2.4 4. KNN Similarity Model

Question: what happened in historical matchups that looked similar?

KNN stands for K-Nearest Neighbors. The model finds the closest historical team-match rows, then averages those neighbors.

Outputs:

  • Expected goals from the average goals scored by similar teams.
  • Loss, draw, and win probabilities from the neighbor outcomes.
  • A flexible comparison against the regression models.

Main issues to check:

  • Similarity depends on which features are included.
  • Features must be standardized so one large-scale number does not dominate.
  • The choice of K controls the smoothness of predictions.
  • KNN can copy noise when the neighbor set is too small.
  • KNN is harder to explain with coefficients because it is case-based, not equation-based.

Open KNN report

2.5 5. Regression Diagnostics

Question: are the regression results stable enough to interpret?

Before the regression results are shown, the workflow checks collinearity, residual patterns, heteroskedasticity, and Poisson overdispersion.

Outputs:

  • VIF values for predictor overlap.
  • Durbin-Watson and lagged residual correlation.
  • Breusch-Pagan and Jarque-Bera diagnostic checks.
  • Poisson dispersion checks for goals.

3 Dictionary

Use this as the quick reference for acronyms and modeling terms that appear across the site.

Term Plain-English Meaning Why It Matters Here
Target variable The thing the model is trying to predict. For the goals model, the target is goals scored. For the result model, the target is win, draw, or loss.
Predictor variable An input used to make the prediction. Team strength, opponent strength, rest, venue, injuries, and odds can all be predictors.
Regression A model that estimates how a target changes with predictors. It gives a structured way to turn soccer information into expected goals or probabilities.
OLS Ordinary Least Squares. A line-fitting method that chooses coefficients by minimizing squared prediction errors. It is the first goals benchmark because it is easy to explain and diagnose.
Poisson regression A regression model for count outcomes. Goals are counts, so Poisson is a natural next model after OLS.
Lambda The expected count in a Poisson model. In the Poisson goals model, lambda is expected team goals.
RMSE Root Mean Squared Error. A typical prediction miss, measured in the same units as the target. If goals RMSE is 1.46, the model is often off by roughly 1 to 2 goals, with big misses punished more. Lower is better.
MAE Mean Absolute Error. The average size of the prediction miss, ignoring direction. Easier to read than RMSE because every miss is weighted evenly. Lower is better.
Coefficient The estimated effect of one predictor, holding the other predictors fixed. A positive strength coefficient means stronger teams are expected to score more, all else equal.
Residual Observed value minus predicted value. Residual plots show whether the model is missing patterns.
VIF Variance Inflation Factor. A check for predictors that repeat the same information. High VIF makes individual regression coefficients harder to trust.
Autocorrelation A pattern where one residual is related to the residual before it. Strong autocorrelation means model errors are not acting independently.
Durbin-Watson A residual autocorrelation check. Values near 2 are usually cleaner. It is a quick test for sequential structure in ordered model errors.
Heteroskedasticity Uneven residual spread across predictor values. It can make OLS standard errors too optimistic.
Overdispersion When a count outcome varies more than a basic Poisson model expects. If goals are overdispersed, a negative-binomial or paired-goal model may handle uncertainty better.
Logit A transformation that converts a probability into log odds. Logistic models use logits so probabilities stay between 0 and 1.
Odds Probability of an event divided by probability of it not happening. Model probabilities can be compared with betting-market odds after converting both to the same scale.
Ordinal logistic regression A logistic model for ordered outcomes. It fits loss < draw < win while producing probabilities for all three outcomes.
Multinomial logistic regression A logistic model for three or more categories without assuming an order. It is the backup if draws behave like their own special category rather than the middle of loss and win.
KNN K-Nearest Neighbors. A model that predicts from similar historical examples. It lets the project compare regression models against a flexible similarity-based method.
Accuracy Share of predictions where the top predicted class matches the actual result. Useful for a quick check, but not enough for betting because probability quality matters too.
Calibration Whether predicted probabilities match real frequencies. If the model says 60% win chance, well-calibrated teams should win about 60% of those cases.
Log loss A probability score that penalizes confident wrong predictions. Better than accuracy when the model outputs win/draw/loss probabilities. Lower is better.
Brier score Average squared error between predicted probabilities and actual outcomes. Another way to judge probability quality. Lower is better.
Test set Data held out from fitting and used only to evaluate performance. It helps estimate how the model performs on games it has not already seen.
Forward-time validation Training on older games and testing on newer games. This is closer to the real task: using past matches to predict future matches.
Overfitting When a model learns noise in past data instead of repeatable signal. A model can look good historically and still fail on new World Cup matches.
Data grain What one row means in a dataset. The main modeling grain is one team in one match. One match creates two team-match rows.
API Application Programming Interface. A structured way to pull data from a provider. API-Football is the live enrichment source once the baseline model is stable.
Ensemble A prediction that combines more than one model. The matchday winner uses ordinal result probabilities plus Poisson score-derived probabilities.
O2.5 Over 2.5 goals. The match has at least 3 total goals. Common soccer totals view derived from the score grid.
BTTS Both teams to score. Another match-level probability derived from expected goals.

This project builds a reproducible data and modeling workflow for the 2026 FIFA World Cup.

The workflow has four layers:

  • Data collection: free match history, Elo-style ratings, 2026 venues, weather, player metadata, and news signals.
  • Data storage: local files and DuckDB tables for modeling.
  • Modeling: interpretable models in R, with Python handling source refreshes and heavier data jobs.
  • Publishing: Quarto renders a shareable GitHub Pages site from the local analysis.

The starting modeling unit is one row per team-match. A single game creates two observations, one for each team.

  if (nrow(metrics) == 0 || !"metric" %in% names(metrics)) {

### 1. Simple OLS Goals Model

**Question:** how many goals should a team score in one match?

The simple goals model starts with team strength difference:

$$
\text{Goals} = \beta_0 + \beta_1\text{Strength Difference} + \epsilon
$$

In words, `beta_1` is the average change in expected goals for a one-unit increase in strength difference.

**Why start here:** OLS is easy to explain, fast to diagnose, and gives a clear benchmark before count models.

**Main issues to check:**

- Goals are counts, so OLS can predict impossible values below 0.
- Soccer scores are low and skewed, so residuals may not look normal.
- Error variance can change by team strength or opponent strength.
- Team-match rows are not fully independent because each match creates two rows.
- Missing roster, injury, rest, venue, and market data can bias coefficients.
:::

::: {.model-card}
### 2. Poisson Goals Model

**Question:** how many goals should a team score when the target is a count?

The Poisson model predicts a positive expected-goals number:

$$
\log(\lambda)
= \beta_0
+ \beta_1\text{Team Elo}
+ \beta_2\text{Opponent Elo}
+ \beta_3\text{Expected Result}
+ \beta_4\text{Match Context}
$$

In words, `lambda` is expected goals. This is a better shape for soccer scoring than OLS because goals cannot be negative.

**Main issues to check:**

- The basic Poisson assumes the mean and variance are tied together.
- Soccer can be overdispersed, meaning the variance may be larger than the mean.
- A negative binomial model may be needed if Poisson is too tight.
- Forward-time validation matters more than random validation for future matches.

<a class="project-button" href="reports/05_poisson_goals_model.html">Open Poisson report</a>
:::

::: {.model-card}
### 3. Three-Outcome Result Model

**Question:** what is the probability of win, draw, or loss?

The first three-outcome model is an ordinal logistic regression because the team result has a natural order:

$$
\text{loss} < \text{draw} < \text{win}
$$

The model uses cumulative logits:

$$
\text{logit}\{P(\text{Result} \leq k)\}
= \theta_k - \eta
$$

with:

$$
\eta = \beta_1\text{Strength Difference}
+ \beta_2\text{Roster Strength}
+ \beta_3\text{Rest}
+ \beta_4\text{Venue}
$$

The output is three probabilities:

$$
P(\text{loss}), \quad P(\text{draw}), \quad P(\text{win})
$$

**Main issues to check:**

- The proportional-odds assumption may be too strong.
- Draws are not simply halfway between loss and win.
- A multinomial logit may fit better if win, draw, and loss behave as separate categories.
- Calibration matters more than raw accuracy for betting comparison.
- Market odds should be the benchmark once API odds are available.
:::

::: {.model-card}
### 4. KNN Similarity Model

**Question:** what happened in historical matchups that looked similar?

KNN stands for **K-Nearest Neighbors**. The model finds the closest historical team-match rows, then averages those neighbors.

**Outputs:**

- Expected goals from the average goals scored by similar teams.
- Loss, draw, and win probabilities from the neighbor outcomes.
- A flexible comparison against the regression models.

**Main issues to check:**

- Similarity depends on which features are included.
- Features must be standardized so one large-scale number does not dominate.
- The choice of `K` controls the smoothness of predictions.
- KNN can copy noise when the neighbor set is too small.
- KNN is harder to explain with coefficients because it is case-based, not equation-based.

<a class="project-button" href="reports/04_knn_similarity_model.html">Open KNN report</a>
:::

::: {.model-card}
### 5. Regression Diagnostics

**Question:** are the regression results stable enough to interpret?

Before the regression results are shown, the workflow checks collinearity, residual patterns, heteroskedasticity, and Poisson overdispersion.

**Outputs:**

- VIF values for predictor overlap.
- Durbin-Watson and lagged residual correlation.
- Breusch-Pagan and Jarque-Bera diagnostic checks.
- Poisson dispersion checks for goals.

## Dictionary

Use this as the quick reference for acronyms and modeling terms that appear across the site.

| Term | Plain-English Meaning | Why It Matters Here |
|---|---|---|
| Target variable | The thing the model is trying to predict. | For the goals model, the target is goals scored. For the result model, the target is win, draw, or loss. |
| Predictor variable | An input used to make the prediction. | Team strength, opponent strength, rest, venue, injuries, and odds can all be predictors. |
| Regression | A model that estimates how a target changes with predictors. | It gives a structured way to turn soccer information into expected goals or probabilities. |
| OLS | Ordinary Least Squares. A line-fitting method that chooses coefficients by minimizing squared prediction errors. | It is the first goals benchmark because it is easy to explain and diagnose. |
| Poisson regression | A regression model for count outcomes. | Goals are counts, so Poisson is a natural next model after OLS. |
| Lambda | The expected count in a Poisson model. | In the Poisson goals model, lambda is expected team goals. |
| RMSE | Root Mean Squared Error. A typical prediction miss, measured in the same units as the target. | If goals RMSE is 1.46, the model is often off by roughly 1 to 2 goals, with big misses punished more. Lower is better. |
| MAE | Mean Absolute Error. The average size of the prediction miss, ignoring direction. | Easier to read than RMSE because every miss is weighted evenly. Lower is better. |
| Coefficient | The estimated effect of one predictor, holding the other predictors fixed. | A positive strength coefficient means stronger teams are expected to score more, all else equal. |
| Residual | Observed value minus predicted value. | Residual plots show whether the model is missing patterns. |
| VIF | Variance Inflation Factor. A check for predictors that repeat the same information. | High VIF makes individual regression coefficients harder to trust. |
| Autocorrelation | A pattern where one residual is related to the residual before it. | Strong autocorrelation means model errors are not acting independently. |
| Durbin-Watson | A residual autocorrelation check. Values near 2 are usually cleaner. | It is a quick test for sequential structure in ordered model errors. |
| Heteroskedasticity | Uneven residual spread across predictor values. | It can make OLS standard errors too optimistic. |
| Overdispersion | When a count outcome varies more than a basic Poisson model expects. | If goals are overdispersed, a negative-binomial or paired-goal model may handle uncertainty better. |
| Logit | A transformation that converts a probability into log odds. | Logistic models use logits so probabilities stay between 0 and 1. |
| Odds | Probability of an event divided by probability of it not happening. | Model probabilities can be compared with betting-market odds after converting both to the same scale. |
| Ordinal logistic regression | A logistic model for ordered outcomes. | It fits `loss < draw < win` while producing probabilities for all three outcomes. |
| Multinomial logistic regression | A logistic model for three or more categories without assuming an order. | It is the backup if draws behave like their own special category rather than the middle of loss and win. |
| KNN | K-Nearest Neighbors. A model that predicts from similar historical examples. | It lets the project compare regression models against a flexible similarity-based method. |
| Accuracy | Share of predictions where the top predicted class matches the actual result. | Useful for a quick check, but not enough for betting because probability quality matters too. |
| Calibration | Whether predicted probabilities match real frequencies. | If the model says 60% win chance, well-calibrated teams should win about 60% of those cases. |
| Log loss | A probability score that penalizes confident wrong predictions. | Better than accuracy when the model outputs win/draw/loss probabilities. Lower is better. |
| Brier score | Average squared error between predicted probabilities and actual outcomes. | Another way to judge probability quality. Lower is better. |
| Test set | Data held out from fitting and used only to evaluate performance. | It helps estimate how the model performs on games it has not already seen. |
| Forward-time validation | Training on older games and testing on newer games. | This is closer to the real task: using past matches to predict future matches. |
| Overfitting | When a model learns noise in past data instead of repeatable signal. | A model can look good historically and still fail on new World Cup matches. |
| Data grain | What one row means in a dataset. | The main modeling grain is one team in one match. One match creates two team-match rows. |
| API | Application Programming Interface. A structured way to pull data from a provider. | API-Football is the live enrichment source once the baseline model is stable. |
| Ensemble | A prediction that combines more than one model. | The matchday winner uses ordinal result probabilities plus Poisson score-derived probabilities. |
| O2.5 | Over 2.5 goals. The match has at least 3 total goals. | Common soccer totals view derived from the score grid. |
| BTTS | Both teams to score. | Another match-level probability derived from expected goals. |

## About

This project builds a reproducible data and modeling workflow for the 2026 FIFA World Cup.

The workflow has four layers:

- **Data collection:** free match history, Elo-style ratings, 2026 venues, weather, player metadata, and news signals.
- **Data storage:** local files and DuckDB tables for modeling.
- **Modeling:** interpretable models in R, with Python handling source refreshes and heavier data jobs.
- **Publishing:** Quarto renders a shareable GitHub Pages site from the local analysis.

## Modeling

The model suite has four core outputs:

- **Goals OLS:** predicts goals scored.
- **Poisson goals:** predicts expected goals as a count.
- **Ordinal logistic:** predicts `loss < draw < win`.
- **KNN similarity:** predicts goals and result probabilities from nearby historical matchups.

The reports show the model target, data grain, math, coefficients, predictions, and diagnostics.

if (nrow(goals_metrics) > 0 && nrow(ordinal_metrics) > 0) {
